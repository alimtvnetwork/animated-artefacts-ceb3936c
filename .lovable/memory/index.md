# Project Memory

## Core
This is a slide-presentation app for **Riseup Asia LLC** (exact spelling — never "Rise Up" or "Riseup-Asia"). Presenter: **MD ALIM UL KARIM**.
**Two themes** ship: `noir-gold` (original — gold `#C9A84C`, cream `#F0D78C`) and `bright-gold` (default — gold `40 88% 50%` v0.36 muted, cream `#fff1d6`). Both on noir bg `#0D0D0D` with ember `#E85D3A`. Switch live via the controller's palette button; persisted with deck on export. NEVER hard-code brand hex in components — always `hsl(var(--gold))`.
Titles: **Ubuntu Bold**, sized via `clamp()` + `max-w-[92vw]` so they never clip. Body: **Inter** (Apple system fallback). Capsules: Inter Semibold.
**Keywords-only content** — never write paragraphs. Presenter narrates; slides are visual anchors. Multiple items → colored capsule labels.
**Spec-first**: before implementing any deck, write JSON+MD specs in `/spec/slides/{deck-name}/NN-name.{json,md}` with images alongside in `images/`. JSON is source of truth at runtime. Always update the spec when behavior changes.
Routing is flat: `/N` (e.g. `/2` = slide 2). URL syncs on every navigation. Deep-linkable.
Controller: **collapsed by default** — single 48px gold-arrow button at ~55% opacity. **Expands to full pill on hover** (prev / N/total / next / grid / presenter / manifest / share / fullscreen). Position: bottom-right.
Slide indicator: click current number → input → Enter jumps to that slide. Share button: full deck OR current slide URL. Manifest button: export/import/reset deck JSON.
Click-reveal slides exist outside linear flow — only shown when triggered from parent item.
Presenter view (`/present`): current + next-up + speaker notes + timer. Sync via BroadcastChannel. **Deck must open the channel ONCE on mount** with handler refs — re-creating it on every render drops messages (see `presenter-sync` memory).
Capsules are vibrant filled buttons (gradient + ink text + glow), not muted tags. `gold`/`ember` capsules pop; `cream`/`ink`/`outline` are quieter variants.
**Hover language is locked deck-wide:** `.lift-hover` (capsules/CTAs) and `.lift-hover-subtle` (chrome/buttons). NO scale/zoom hovers — only translateY + soft drop shadow. Defined in `index.css`. Reuse, never invent.
**Branded QR is a PNG, not generated:** `<BrandedQR asset="<slug>" />` renders a designer-authored PNG from `src/assets/brand/` at native 1:1, white tile + ink modules. Never red on black. Register slugs in `BrandedQR.tsx`.
**No Lovable branding** anywhere — no logo, favicon, og tags, meta references.
**NEVER enable the top BrandStrip banner** (logo+tagline strip above BrandHeader). User has rejected it repeatedly. Keep `deck.brandStrip` OUT of deck.json and do not re-add per-slide. The standard `BrandHeader` (Riseup logo + slide counter) is the only top chrome allowed.
Animations: variety required. Transitions: FadeIn, SlideIn, PushIn, PushLeft, PushRight. Text: Bounce, FadeIn, SlideUp, Stagger. Respect `prefers-reduced-motion`.
**StepsChain3DSlide (slide 4) is CLICK-ONLY.** Never add hover effects on step cards (no glow ring, brightness, lift, marker change). `cursor:pointer` is the only allowed hover affordance — the click activation animation IS the feedback. See [no-hover-on-steps-chain-3d](mem://constraints/no-hover-on-steps-chain-3d).
Slide types: TitleSlide, KeywordSlide, CapsuleListSlide, StepTimelineSlide, FocusTimelineSlide, AdvanceStepSlide, ImageSlide, QrMeetingSlide, ClickRevealSlide, SectionDividerSlide, TableSlide, CodeBlockSlide, BoxDiagramSlide, ERDiagramSlide, MetricGridSlide, LayoutSlide.
Brand assets: `src/assets/brand/` (logo, alim-presenter.png). Use semantic tokens from index.css — never raw hex in components.
**Milestone log lives in `readme.md`** (NOT readme.txt — that file no longer exists). Append `let's start now {date} {time}` + `vX.Y.Z — summary` + bullet list of file changes for every release. Malaysia UTC+8.
**Light themes (github-light, macos-sonoma) MUST carry a soft cool tint + radial body wash — never flat #ffffff.** Slide types must NEVER hardcode `bg-ink`/`bg-white`/literal colors on root containers — always inherit `hsl(var(--background))`. See [light theme bg](mem://design/light-theme-bg).
**Narrow Idea Per Slide:** every slide communicates exactly ONE narrow idea. If it teaches two, split it. Keywords only. Presenter narrates. Density is a defect. KeywordSlide ≤6 capsules; DataTableSlide ≤5 cols; NumberCalloutSlide = 1 number; EquationSlide = 1 equation; DatabaseDiagramSlide = 1 schema concept. See [slide narrow idea](mem://design/slide-narrow-idea).
**Capsules MUST use `.capsule-{tone}` classNames — NEVER inline `style.background`/`color` referencing brand tokens (`--gold/--ember/--cream/--ink/--white`).** Those tokens change meaning between dark/light themes (e.g. `--cream` is repurposed to ink on paper-ink/github-light), so inline styles silently break contrast on light themes. Use `.capsule-meta` for time tags. See [light-theme capsule fg rule](mem://design/light-theme-capsule-fg-rule), `updates/spec/15+16`.
